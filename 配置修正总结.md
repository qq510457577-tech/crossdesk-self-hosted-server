## 参考文档
基于CrossDesk官方`file-transfer`分支的最新文档进行修正：

## 发现的主要差异

### 1. Docker镜像版本
- **原配置**: `v1.0.0`
- **修正后**: `v1.1.6` (最新稳定版)

### 2. 服务端口
- **原配置**: `8443`
- **修正后**: `9099` (文档推荐端口)

### 3. 卷挂载路径
- **原配置**: 自定义路径 `/root/crossdesk-server/{certs,db,logs}`
- **修正后**: 标准路径 `/var/lib/crossdesk` 和 `/var/log/crossdesk`

### 4. 证书管理
- **原配置**: 手动生成证书并挂载
- **修正后**: 容器首次启动时自动生成证书
- **证书文件名**: `api.crossdesk.cn_*` (原为 `crossdesk.cn_*`)

## 修正步骤

### 1. 停止旧容器
```bash
cd /root/crossdesk-server && docker compose down
```

### 2. 创建标准目录
```bash
sudo mkdir -p /var/lib/crossdesk /var/log/crossdesk
sudo chown -R $(id -u):$(id -g) /var/lib/crossdesk /var/log/crossdesk
```

### 3. 拉取新镜像
```bash
docker pull crossdesk/crossdesk-server:v1.1.6
```

### 4. 更新docker-compose.yml
主要变更：
- 镜像版本: `v1.0.0` → `v1.1.6`
- 服务端口: `8443` → `9099`
- 卷挂载: 自定义路径 → 标准路径

### 5. 启动新服务
```bash
cd /root/crossdesk-server && docker compose up -d
```

## 证书信息

### 自动生成的证书
- **根证书**: `api.crossdesk.cn_root.crt` (4096位RSA)
- **服务器证书**: `api.crossdesk.cn_bundle.crt` (2048位RSA)
- **服务器私钥**: `api.crossdesk.cn.key`

### 证书特性
- **有效期**: 2026-04-14 至 2036-04-11 (10年)
- **算法**: SHA256WithRSAEncryption
- **主题**: CN=43.133.192.17 (服务器IP地址)
- **自动生成**: 容器首次启动时自动生成

## 客户端配置更新

### 新的配置参数
- **服务器地址**: `43.133.192.17`
- **信令服务端口**: `9099` (原8443)
- **中继服务端口**: `3478` (不变)

### 根证书文件
- **文件名**: `api.crossdesk.cn_root.crt` (原 `crossdesk.cn_root.crt`)
- **位置**: `/var/lib/crossdesk/certs/api.crossdesk.cn_root.crt`

### 客户端安装命令（根据文档）

#### Windows
```powershell
certutil -addstore "Root" "C:\path\to\api.crossdesk.cn_root.crt"
```

#### Linux
```bash
sudo cp /path/to/api.crossdesk.cn_root.crt /usr/local/share/ca-certificates/api.crossdesk.cn_root.crt
sudo update-ca-certificates
```

#### macOS
```bash
sudo security add-trusted-cert -d -r trustRoot -k /Library/Keychains/System.keychain /path/to/api.crossdesk.cn_root.crt
```
