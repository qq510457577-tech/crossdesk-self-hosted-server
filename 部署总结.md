# CrossDesk 自托管服务器部署总结

## 部署概述
基于CrossDesk官方文档，在本地Docker环境中成功部署了自托管的CrossDesk服务器。

## 部署时间

## 部署步骤

### 1. 环境准备
- 创建目录结构：`certs/`, `db/`, `logs/`
- 安装Docker和Docker Compose

### 2. 证书生成
- 使用OpenSSL生成自签名证书
- 服务器IP：43.133.192.17
- 生成文件：
  - `crossdesk.cn_root.crt` (根证书，客户端使用)
  - `crossdesk.cn.key` (服务器私钥)
  - `crossdesk.cn_bundle.crt` (服务器证书)

### 3. Docker容器部署
- 使用官方镜像：`crossdesk/crossdesk-server:v1.0.0`
- 网络模式：host (便于端口映射)
- 环境变量配置：
  - EXTERNAL_IP=43.133.192.17
  - INTERNAL_IP=172.17.0.1
  - CROSSDESK_SERVER_PORT=8443
  - COTURN_PORT=3478
  - MIN_PORT=50000
  - MAX_PORT=60000

### 4. 服务启动
- 使用Docker Compose启动服务
- 容器名称：`crossdesk_server`
- 重启策略：unless-stopped

## 服务配置

### 端口配置
- **8443/tcp**: CrossDesk信令服务器
- **3478/tcp**: COTURN服务
- **3478/udp**: COTURN服务
- **50000-60000/udp**: COTURN端口范围

### 目录挂载
- `./certs` → `/crossdesk-server/certs` (证书文件)
- `./db` → `/crossdesk-server/db` (设备管理数据库)
- `./logs` → `/crossdesk-server/logs` (日志文件)
