- **端口范围**: 50000-60000 (UDP)

## 证书配置

### 1. 下载根证书
根证书文件已生成在服务器上，您需要下载它到客户端：
- **根证书文件**: `crossdesk.cn_root.crt`
- **位置**: `/root/crossdesk-server/crossdesk.cn_root.crt`

您可以通过以下方式获取：
```bash
# 从服务器下载（如果您有访问权限）
scp root@43.133.192.17:/root/crossdesk-server/crossdesk.cn_root.crt .
```

### 2. 客户端配置步骤

#### Windows/macOS/Linux 客户端：
1. **安装CrossDesk客户端**
   - 从官方网站下载：https://www.crossdesk.cn/

2. **配置自托管服务器**
   - 打开CrossDesk客户端
   - 点击右上角设置图标进入设置页面
   - 点击"自托管服务器配置"

3. **加载根证书**
   - 在证书文件路径选择框中找到 `crossdesk.cn_root.crt` 的存放路径
   - 选中 `crossdesk.cn_root.crt`，点击确认

4. **服务器配置**
   - 勾选"使用自托管服务器配置"
   - 填写服务器信息：
     - **服务器地址**: 43.133.192.17
     - **服务器端口**: 8443
     - **中继服务端口**: 3478
   - 点击确认使配置生效

## 防火墙配置

服务器需要开放以下端口：

### TCP端口：
- 8443 - CrossDesk信令服务器
- 3478 - COTURN服务

### UDP端口：
- 3478 - COTURN服务
- 50000-60000 - COTURN端口范围

如果您使用云服务器，请确保安全组/防火墙规则允许这些端口。

## 测试连接

### 1. 服务器状态检查
```bash
# 检查容器状态
docker ps | grep crossdesk

# 查看服务器日志
docker logs crossdesk_server --tail 10

# 检查端口监听
netstat -tlnp | grep -E "8443|3478"
```

### 2. 客户端连接测试
1. 在客户端输入对端ID
2. 点击连接按钮
3. 如果远端设置了密码，输入正确的连接密码

## 故障排除

### 常见问题：

#### 1. 客户端显示"未连接服务器"
- 检查服务器地址和端口是否正确
- 确认根证书已正确加载
- 检查防火墙设置

#### 2. 连接超时
- 确认服务器端口已开放
- 检查网络连接
- 验证证书有效性

#### 3. 音视频质量差
- 确保UDP端口范围已正确开放
- 检查网络带宽
- 调整客户端视频设置

## 管理命令

### 服务器管理：
```bash
# 启动服务器
cd /root/crossdesk-server && docker compose up -d

# 停止服务器
cd /root/crossdesk-server && docker compose down

# 重启服务器
cd /root/crossdesk-server && docker compose restart

# 查看日志
docker logs crossdesk_server -f

# 进入容器
docker exec -it crossdesk_server /bin/bash
```

### 证书管理：
```bash
# 重新生成证书（如果需要）
```
